Twitter Mining Project

The purpose of this project is to perform an analysis of tweets. The tweets were mined with R from twitter.

The first step towards a tweet analysis is loading the required libraries.

Searching the Twitter.

searchTwitter() function will mine the tweets for us. In this case we are searching Twitter for the topic “morning+coffee” and we ask searchTwitter() function to bring us the 500 most recent tweets written in English. The output of this function is a list.

tweets <- searchTwitter("morning+coffee", n = 500 , lang = "en" , resultType = "recent")
class(tweets) 
## [1] "list"

with the following function we transform the tweets list to a character vector.

tweets_text <- sapply(tweets, function(x) x$getText())
class(tweets_text)
## [1] "character"

Then with the use of tm package we transform the character vector we created above to a Corpus that will help us perform our analysis.

tweet_corpus <- Corpus(VectorSource(tweets_text))
tweet_corpus
## <<VCorpus>>
## Metadata:  corpus specific: 0, document level (indexed): 0
## Content:  documents: 500

Cleaning data.

The next step in the process is to remove any unwanted terms and character. In this particular case we use tm_map() function 6 times and we do the following: a) remove any non-english word and characters. b) make all characters lowercase. c) remove commonly used words with stopwords function. d) remove any white space. e) remove urls and f) remove numbers.

removeNumPunct <- function(x) gsub("[^[:alpha:][:space:]]*", "", x)
tweet_corpus_clean <- tm_map(tweet_corpus, content_transformer(removeNumPunct))
tweet_corpus_clean <- tm_map(tweet_corpus_clean, content_transformer(tolower))
tweet_corpus_clean <- tm_map(tweet_corpus_clean, removeWords, stopwords("en"))
tweet_corpus_clean <- tm_map(tweet_corpus_clean, stripWhitespace)
removeURL <- function(x) gsub("http[^[:space:]]*", "", x)
tweet_corpus_clean <- tm_map(tweet_corpus_clean, content_transformer(removeURL)) 
tweet_corpus_clean <- tm_map(tweet_corpus_clean, removeNumbers)

WordClouds

After having removed all of the above we can use the wordcloud() function from tm package in order to create the wordcloud. Below there are two wordclouds, the first one is simply a function call on the clean corpus, and the second one has more parameters in it, for example we have deactivated the random.order function by setting it to FALSE, we have set a limit of 500 to the number of words that will appear in the wordcloud, we have the range of the character scale between 3 and 0.5 and finally we have set the colors parameter to “rainbow(50)” which means that we want to use 50 colors of the rainbow as color for the wordcloud.

wordcloud(tweet_corpus_clean) 

wordcloud(tweet_corpus_clean, random.order = F, max.words = 500, scale = c(3,0.5), colors = rainbow(50))

1. Hierarchical clustering.

Performing Hierarchical clustering is a 3-step process:

1.1 First calculate distance between words & then cluster them according to similarity.

library(cluster)   
s <- dist(t(dtmss), method="euclidian")

1.2 Create the model

fit <- hclust(d=s, method="ward.D2")   
fit
##

## hclust(d = s, method = "ward.D2")

1.3 Plot the Model.

plot(fit, hang=-1)

1.4 Asking R to identify the clusters and Create the borders.

plot.new()
plot(fit, hang=-1)
groups <- cutree(fit, k=5)   
rect.hclust(fit, k=5, border="red")
